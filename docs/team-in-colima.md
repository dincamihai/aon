# Team in Colima — running an `aon` team inside the sandbox

Operator-facing guide for running a per-team agent fleet inside the
colima/AppArmor sandbox, with the operator on macOS host and one
interactive Claude Code session per role inside the VM. Pairs with
[`docs/sandbox.md`](sandbox.md) (sandbox internals) and the cmd-gate
ADRs ([001](adr/001-cmd-gate-layered-safety-gate.md),
[002](adr/002-self-evolving-classifier-prompt.md)).

> TL;DR. `cd ~/Repos/<team> && AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux --restart`
> One tmux session, one pane per role. Each pane = a live `claude`
> running as `ta-worker-<role>` inside the VM, persistent across ssh
> drops (via `dtach`), wired to the host's NATS broker, gated by
> cmd-gate's PreToolUse hook with sandbox-aware bypass.

## Architecture

```
Mac host                              colima VM (profile = "aon")
─────────                             ─────────────────────────
 nats-server :4222 (host)             agents (claude TUI)
   ├─ sysadmin.creds (host-local)       ├─ ta-worker-rona     dtach @ /tmp/aon-rona.sock
   └─ <role>.creds                      ├─ ta-worker-tim      dtach @ /tmp/aon-tim.sock
                                        ├─ ta-worker-sun      dtach @ /tmp/aon-sun.sock
 tmux session "<team>"                  ├─ ta-worker-mid      …
   ├─ window "team":                    ├─ ta-worker-joana    …
   │    panes attach via                └─ ta-worker-ari      …
   │    ssh -t … dtach -a sock                  AppArmor enforce
   │                                            UIDs isolated, $HOME 0700
   └─ window "security":                        per-role .claude.json (no sysadmin)
        aon security watch                      per-role NATS .creds
        (sysadmin creds, host)                  agents → host.lima.internal:4222

 slack-mcp-rtm (launchd)              /opt/aon-mcp/venv/bin/aon-mcp
   └─ events.jsonl                    /opt/board-tui/venv/bin/board-tui-mcp
 slack-bridge-sun (launchd)
   └─ pub agents.sun.inbox → NATS
```

Three trust zones, three boundaries:

| Zone | Lives on | Holds |
|---|---|---|
| Operator | macOS host | sysadmin creds, OAuth account, gate TUI |
| VM (root) | Linux VM | systemd units, AppArmor profiles, /etc/team-alpha |
| Agent | VM agent UID | own worktree, own creds, own .claude.json |

Each agent UID can read **only** its own creds + worktree. AppArmor +
DAC + per-role NATS ACL enforce.

## Components

### Host

| Path | Purpose |
|---|---|
| `~/Repos/ai-over-nats/scripts/sandbox/aon-tmux.sh` | engine launcher (team-agnostic) |
| `~/Repos/<team>/scripts/aon-tmux` | thin shim, delegates to engine, generated by `aon admin init` |
| `~/Repos/<team>/aon.toml` | roster + cmd-gate config |
| `~/Repos/board-tui/` | board-tui source — must be present; install-in-vm.sh builds a Linux venv from it |
| `~/.aon/teams/<team>/creds/<role>.creds` | per-role NATS JWT — only that role's creds copied to VM |
| `~/.aon/teams/<team>/creds/sysadmin.creds` | operator NATS identity — **never** copied to VM |
| `~/.config/slack-mcp/config.toml` | slack MCP auth — pushed to `AON_SLACK_ROLES` (default sun) on each `--restart` |
| `~/.config/slack-mcp/events.jsonl` | Slack event sink written by `slack-mcp-rtm` (launchd) |
| `~/.aon/colima-aon.ssh-config` | rendered colima ssh-config for `ssh -F` PTY use |
| `~/Library/LaunchAgents/com.user.slack-mcp-rtm.plist` | keeps `slack-mcp-rtm` running across reboots |
| `~/Library/LaunchAgents/com.user.slack-bridge-sun.plist` | keeps `slack-bridge.sh sun` running across reboots |

### VM

| Path | Purpose |
|---|---|
| `/work/workers/<role>/` | role's git worktree (cloned `--shared` from host's read-only mount on first start) |
| `/work/workers/<role>/.claude/settings.local.json` | per-role statusline + sonnet default |
| `/work/workers/<role>/.mcp.json` | per-role MCP config: aon + aon-board (+ slack for `AON_SLACK_ROLES`) |
| `/var/lib/team-alpha/workers/<role>/` | role's `$HOME` (0700) |
| `/var/lib/team-alpha/workers/<role>/.claude.json` | OAuth account metadata (sanitized: `installMethod=global-npm`, `projects` stripped) |
| `/var/lib/team-alpha/workers/<role>/.claude/.credentials.json` | OAuth tokens (copied from `ta-claude-auth` user — see `aon admin claude-login`) |
| `/var/lib/ta-claude-auth/` | dedicated VM user holding the shared claude OAuth login |
| `/usr/local/bin/aon` | wrapper exec'ing host-mounted `$TA_HARNESS/bin/aon` |
| `/usr/local/bin/aon-mcp` | symlink → `/opt/aon-mcp/venv/bin/aon-mcp` (Linux venv built from host source) |
| `/usr/local/bin/board-tui-mcp` | symlink → `/opt/board-tui/venv/bin/board-tui-mcp` (Linux venv built from `~/Repos/board-tui`) |
| `/usr/local/bin/nats`, `/usr/local/bin/uvx` | downloaded binaries (apt only ships `nats-server`, no `uvx`) |
| `/etc/team-alpha/creds/<role>.creds` | NATS creds, 0600, ACL grants only that UID `r` |
| `/etc/team-alpha/env` | shared env: `AON_NATS_URL`, `TA_HARNESS`, `TA_SLACK_EVENTS_DIR`, etc. |
| `/etc/team-alpha/bypass` | sandbox bypass marker (root-only — agent denied write by AppArmor) |
| `/tmp/aon-<role>.sock` | dtach socket; one live `claude` process bound to it |

### NATS topology (gate flow)

```
agent:  evt.coord-in.gate-request.<role>.<id>   ──→  operator
operator: evt.coord-out.gate-reply.<role>.<id>  ──→  agent
audit:  evt.security.gate.<role>                ──→  sysadmin (TUI / shadow)
```

Per-role ACL (auth template):
- `generalist` / `specialist`: pub on `gate-request.<own-role>.>`, sub on `gate-reply.<own-role>.>`. **Cannot** pub on reply path.
- `manager`: same plus other manager perms.
- `sysadmin`: wildcard `>` for both — operator's identity.

Agent **cannot** self-approve, **cannot** impersonate peer.

## Daily flow

### Start the team (host terminal)

```bash
cd ~/Repos/<team>
AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux --restart
```

`--restart`: kills any stale dtach sessions in VM + tears down old host tmux. Fresh state every time. Drop the flag for normal "attach to running" runs.

What it does:
1. Render colima ssh-config to `~/.aon/colima-<profile>.ssh-config`.
2. For each role from the team's `aon.toml` roster:
   - Auto-create `ta-worker-<role>` user + `/work/workers/<role>` if missing (via `add-worker.sh`).
   - Push role's NATS creds to `/etc/team-alpha/creds/<role>.creds` (0600 + ACL).
   - `AON_SHARE_CLAUDE_AUTH=1`: copy OAuth tokens from `ta-claude-auth` VM user into role's `$HOME/.claude/`.
   - Run `start-agent-in-vm.sh <role>`:
     - Self-heals `aon-mcp` and `board-tui` Linux venvs if missing (idempotent).
     - First start: clones host's team repo (shared, hardlinks) into `/work/workers/<role>/`.
     - Writes `.claude/settings.local.json` with statusline badge + `model=sonnet`.
     - Writes `.mcp.json` with `aon` + `aon-board` servers (+ `slack` for `AON_SLACK_ROLES`).
     - Spawns claude under `dtach -n /tmp/aon-<role>.sock` with full env (`AON_ROLE`, `AON_ROLE_KIND`, `AON_ROLE_DOMAIN`, `AON_TEAM`, `AON_KV_BUCKET`, `AON_NATS_URL`, `AON_CREDS`, `AON_MCP_BIN`, `BOARD_TUI_MCP_BIN`, `TERM=xterm-256color`, `COLORTERM=truecolor`).
     - Verifies dtach is alive after 0.5 s; bails loudly if claude crashed at startup.
3. Create tmux session `<team>` with two windows:
   - **`team`** — tiled panes, one per role. Each pane runs `ssh -F <conf> -t <host> sudo -E -u ta-worker-<r> env TERM=$TERM dtach -a /tmp/aon-<r>.sock`. Detach a pane (close the ssh) → claude keeps running in VM.
   - **`security`** — single pane running `aon security watch` (textual TUI, sysadmin creds, host).
4. Attach to the session.

### Switch panes

| Keys | Action |
|---|---|
| `Ctrl-b 0` / `Ctrl-b 1` | jump to window team / security |
| `Ctrl-b o` | cycle pane within current window |
| `Ctrl-b ←/→/↑/↓` | move to neighbor pane |
| `Ctrl-b z` | zoom current pane |
| `Ctrl-b d` | detach session — agents keep running |

### Approve gate-asks

Window `security` shows the textual TUI. When an agent in the team window publishes a `gate-request`, it appears in the TUI:

| Key | Action |
|---|---|
| `↑`/`↓`, `j`/`k` | navigate pending list |
| `y`, `Enter` | approve |
| `n` | deny |
| `r` | deny + reason (modal) |
| `q` | quit |

The approve / deny round-trips on NATS; the agent's blocked Bash unfreezes in the team-window pane.

### Detach for the night

```
Ctrl-b d
```

Agents stay alive in VM (dtach holds the claude processes; sockets persist). NATS broker on host stays running. Reattach next session:

```bash
cd ~/Repos/<team>
AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux
```

(no `--restart` → ssh PTY reattaches to existing dtach sockets, conversations resume mid-thought.)

### Stop the team

```bash
./scripts/aon-tmux --restart                 # cleans state but immediately re-launches
```

Or to truly stop:

```bash
ssh -F ~/.aon/colima-aon.ssh-config -o ControlPath=none colima-aon \
  'sudo pkill -9 -f "dtach -n /tmp/aon-"; sudo rm -f /tmp/aon-*.sock'
tmux kill-session -t <team>
```

To stop everything including the VM:

```bash
colima stop --profile aon
```

VM persists between stops; restart with `colima start --profile aon`.

## One-time setup — from scratch

### 0. Host prerequisites (macOS)

```bash
brew install colima lima docker          # vz hypervisor + docker CLI
brew install bash                        # bash >= 4 for aon
brew install jq nats-io/nats-tools/nats  # nats CLI for `aon security watch`
brew install nats-io/nats-tools/nsc      # only if you'll re-mint creds
brew install coreutils                   # gtimeout, used by nsc-smoke
```

Clone engine + board-tui (must be siblings under `~/Repos/`):

```bash
mkdir -p ~/Repos && cd ~/Repos
git clone <ai-over-nats-url> ai-over-nats
git clone https://github.com/dincamihai/board-tui.git board-tui  # needed for board MCP venv in VM
```

Create team:

```bash
cd ~/Repos
pipx install --editable ./ai-over-nats        # exposes `aon` on PATH
mkdir <team> && cd <team>
aon admin init                                # bootstrap aon.toml
aon admin onboard rona                        # add a role (repeat per role)
```

NATS broker on host:

```bash
aon admin nats up                             # docker-compose nats:4222
```

Bootstrap NATS streams + KV (required before first agent start):

```bash
cd ~/Repos/<team>
aon admin reinit                              # mint creds + streams + KV bucket
```

This creates the `<team>-state` KV bucket that the aon MCP server looks for. Without it the MCP server exits at startup with `KV bucket '<team>-state' not found`.

### 1. Bring up VM (one-shot)

```bash
cd ~/Repos/ai-over-nats
TA_PROFILE=aon \
TA_HARNESS=$PWD \
TA_REPOS=$HOME/Repos \
TA_EXTERNAL_NATS=auto \
  bash scripts/sandbox/colima-up.sh
```

What this provisions inside the VM:
- apt: `acl apparmor auditd ca-certificates curl git jq nftables unzip nodejs npm nats-server systemd`
- nats CLI (download from github releases — apt only ships server)
- uv / uvx (download from github releases — needed by Python MCP servers)
- `/usr/local/bin/aon` wrapper that exec's the host-mounted `bin/aon`
- `/opt/aon-mcp/venv` — Linux venv for aon MCP server (built from `$TA_HARNESS/mcp-server`)
- `/opt/board-tui/venv` — Linux venv for board-tui MCP (built from `$(dirname $TA_HARNESS)/board-tui`)
- AppArmor profiles in enforce mode
- systemd units: `team-alpha-coord`, `team-alpha-worker@<role>`, `team-alpha-slack-bridge@<role>`
- `/etc/team-alpha/env` with `AON_NATS_URL=nats://host.lima.internal:4222`

Mounts (declared in `colima-up.sh`):

| Env var | Host path | VM mode | Purpose |
|---|---|---|---|
| `TA_HARNESS` | `~/Repos/ai-over-nats` | RO | engine source, aon wrapper, MCP venv source |
| `TA_REPOS` | `~/Repos` | RO | team worktrees, slack-mcp + board-tui source |
| `TA_AON_BOARD` | `~/aon-board` | RW | board MCP state — auto-detected if dir exists |
| `TA_SLACK_EVENTS_DIR` | `~/.config/slack-mcp` | RO | Slack events JSONL for in-VM bridge (auto-detected) |
| `TA_LOCAL_APPARMOR` | `~/.team-alpha/apparmor` | RO | personal AppArmor overrides |

`TA_EXTERNAL_NATS=auto` sets `AON_NATS_URL=nats://host.lima.internal:4222` so VM agents reach the host broker; in-VM `team-alpha-nats` unit is masked. Operator's `sysadmin.creds` stays on host.

Verify:

```bash
colima ssh --profile aon -- bash -c 'which aon nats uvx board-tui-mcp; mount | grep Users'
```

### 2. Install per-team launcher shim (per team)

```bash
cd ~/Repos/<team>
aon admin init                                    # idempotent; drops scripts/aon-tmux shim
git add scripts/aon-tmux && git commit -m "..."   # share with team
```

### 3. Seed claude OAuth in VM (one-time per VM)

Avoids login-per-role + the macOS Keychain detour. Logs in once as a dedicated VM user; aon-tmux copies the resulting tokens into each role's home on each `--restart`.

```bash
aon admin claude-login                            # interactive — complete OAuth in VM, then /exit
```

What it does:
1. Creates `ta-claude-auth` user in the VM (if missing).
2. Opens an interactive ssh PTY into the VM running `claude` as that user.
3. Complete the OAuth `/login` flow, then `/exit`.
4. Verifies `~/.claude/.credentials.json` was written.

Re-run only when the OAuth token rotates or the VM is recreated.

### 4. Slack MCP + bridge (optional, default role: sun)

Two components: **slack-mcp** (MCP tool server, runs inside claude) and **slack-mcp-rtm + slack-bridge** (live message delivery to sun's inbox via NATS).

#### 4a. Auth config

```bash
mkdir -p ~/.config/slack-mcp
cp ~/Repos/slack-mcp/config.example.toml ~/.config/slack-mcp/config.toml
# edit config.toml — add xoxc/xoxd tokens and channel list
```

`aon-tmux.sh` `share_slack_config()` scp's this into each role's home in the VM on every `--restart`. To grant slack to other roles:

```bash
AON_SLACK_ROLES="sun mid" ./scripts/aon-tmux --restart
```

#### 4b. Live message bridge (launchd — macOS)

`slack-mcp-rtm` polls Slack every 8 s and writes events to `~/.config/slack-mcp/events.jsonl`. `slack-bridge.sh sun` tails that file and publishes each message to `agents.sun.inbox` on NATS. Both need launchd entries to survive reboots and session ends.

```bash
# Install rtm daemon (source: ~/Repos/slack-mcp/launchd/)
cp ~/Repos/slack-mcp/launchd/com.user.slack-mcp-rtm.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.user.slack-mcp-rtm.plist

# Install bridge daemon (source: ~/Repos/workers/launchd/)
cp ~/Repos/workers/launchd/com.user.slack-bridge-sun.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.user.slack-bridge-sun.plist
```

Verify both running:

```bash
launchctl list | grep slack
# should show com.user.slack-mcp-rtm and com.user.slack-bridge-sun with PIDs
```

The bridge publishes to `nats://localhost:4222` (host broker). VM's sun agent connects to the same broker via `host.lima.internal:4222` — no extra wiring needed.

### 5. Configure cmd-gate evolve (optional)

In the team's `aon.toml`:

```toml
[security.cmd_gate.evolve]
backend_provider = "ollama"
model            = "qwen3.5:9b"
budget_usd       = 0
```

For Anthropic judge instead, set `backend_provider = "anthropic"` and `ANTHROPIC_API_KEY` in env.

### 6. Personal AppArmor overrides (optional)

See [`docs/sandbox.md` § Personal AppArmor overrides](sandbox.md). Drop additional allow/deny rules in `~/.team-alpha/apparmor/` and re-run `colima-up.sh`.

### 7. First launch

```bash
cd ~/Repos/<team>
AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux --restart
```

In each pane: `/mcp` lists `aon`, `aon-board` (and `slack` for sun). Statusline shows `<role> - <kind>`. A2A accept-loop line in each pane's MCP startup confirms NATS is connected.

## Per-role MCP wiring

`start-agent-in-vm.sh` writes `$work/.mcp.json` after cloning the team repo. All roles get:

| Server | Command | Notes |
|---|---|---|
| `aon` | `aon mcp-server aon` | task board, DM, A2A, team state via NATS |
| `aon-board` | `aon mcp-server board` | board-tui card manager; `BOARD_TASKS_DIR=~/aon-board` |

Roles in `$AON_SLACK_ROLES` (default `sun`) additionally get:

| Server | Command | Notes |
|---|---|---|
| `slack` | `uvx --from /Users/.../slack-mcp slack-mcp` | Slack read/write/DM |

`aon-mcp` and `board-tui-mcp` are Linux binaries built in VM venvs — the host's `.venv` is macOS-built and can't run inside the VM. The binaries are at `/usr/local/bin/aon-mcp` and `/usr/local/bin/board-tui-mcp`; paths passed via `AON_MCP_BIN` + `BOARD_TUI_MCP_BIN` env.

To customize, edit the `mcp_extra` block in `start-agent-in-vm.sh`. Per-role conditional MCPs follow the same pattern.

## Trust model recap

The gate is **intent gating** — argv-level. Defense in depth:

| Layer | Stops |
|---|---|
| `cmd-gate` deny.regex hard floor | `rm -rf /`, `git push --force`, `terraform destroy`, etc. — even with bypass |
| `cmd-gate` classifier | hidden flags, base64 obfuscation, char-code SQL, etc. |
| `cmd-gate` operator-ask | borderline argv — operator approves via TUI |
| AppArmor `team-alpha-worker` | per-UID FS scope, syscall filter, deny `/etc/`, deny peer worktrees |
| DAC (Linux UID + 0700 dirs) | each agent reads/writes only own home + worktree |
| NATS ACL (per-role JWT) | agent can't impersonate peer or self-approve |
| host broker outside VM | sysadmin creds untouchable from VM (kernel boundary) |
| colima VM (vz/aarch64) | kernel boundary; agent escape doesn't reach host |

What this **doesn't** defend:

- Anthropic API session: VM-side `ta-claude-auth` OAuth shared across roles → all agents bill to one account; concurrent session limits may apply.
- Network egress: agent in VM can reach any host its iptables allows. Add `nft` rules per [`docs/sandbox.md` § What this does not do](sandbox.md).
- Kernel exploits: VM is the last line. Agents are kernel-isolated from host, but exploit + escape gets you a VM-level breach (host still safe).

## Configuration knobs

| Env / file | Effect |
|---|---|
| `AON_SHARE_CLAUDE_AUTH=1` | copy claude OAuth from `ta-claude-auth` VM user to each role (default on; set =0 to disable) |
| `AON_SLACK_ROLES="sun mid"` | space-separated list of roles that get slack MCP (default: `sun`) |
| `AON_COLIMA_PROFILE=<name>` | colima profile (default `aon`) |
| `AON_TMUX_SESSION=<name>` | tmux session name (default = team basename) |

| `~/.team-alpha/apparmor/{base,coord,worker}` | personal AppArmor overrides synced into VM |

| `aon.toml [security.cmd_gate]` | classifier model, timeouts, fallback |
| `aon.toml [security.cmd_gate.evolve]` | judge backend + model + budget |

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `can't find window: team` | dtach socket exists but claude crashed | `./scripts/aon-tmux --restart` (start-agent now detects + clears orphans) |
| `Not logged in · Run /login` in every pane | claude OAuth not seeded in VM | run `aon admin claude-login` then `AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux --restart` |
| `/mcp` shows `aon` as failed / disconnected | `AON_KV_BUCKET` wrong or KV not created | check `/etc/team-alpha/env` has `TA_SLACK_EVENTS_DIR`; run `aon admin reinit` from team repo on host |
| `KV bucket 'team-state' not found` in MCP log | agents started before `aon admin reinit`; or `AON_KV_BUCKET` unset | run `aon admin reinit` from `~/Repos/<team>`; restart agents |
| `board-tui-mcp not on PATH` | board-tui venv not built | ensure `~/Repos/board-tui` exists on host (mounted RO); run `start-agent-in-vm.sh <role>` — self-heal builds it |
| Statusline shows `? - ?` | `AON_ROLE_KIND` not resolved from aon.toml | ensure work dir contains aon.toml; `--restart` to re-run start-agent |
| Port 4222 conflict: `NATS authorization violation` | stale colima ssh mux has a port-forward on `localhost:4222` (IPv4) shadowing nats-server | `lsof -iTCP:4222 \| grep ssh` — kill that PID; run `aon admin reinit` |
| Mono colors (kitty terminal) | `kitty term=xterm-kitty` doesn't propagate | set `term xterm-256color` in `~/.config/kitty/kitty.conf` |
| `unknown shorthand flag: t in -t` | tried `colima ssh -t` — colima ssh has no PTY flag | use `ssh -F ~/.aon/colima-<profile>.ssh-config -t colima-<profile>` |
| `Inappropriate ioctl for device` from claude | TERM=unknown bled through | `--restart`; the dtach env now hardcodes TERM=xterm-256color |
| `Session open refused by peer` / `exit 255` | stale ssh control socket | `rm ~/.colima/_lima/colima-aon/ssh.sock`; `aon-tmux` does this automatically each run |
| `agents/ missing` from `aon doctor` | A2A discovery cards never populated | `mkdir ~/Repos/<team>/agents` (cosmetic — only matters if you use A2A skill routing) |
| Gate denies a legitimate command | `deny.regex` is hard-floor (irreversible) | use `~/.aon/security/allow.local.regex` to whitelist the pattern; or `aon security bypass on` for short-lived dev (still respects deny.regex) |
| Operator TUI shows nothing on agent's gate-request | per-role NATS perms stale | re-mint role JWT: `nsc delete user --account <team> --name <role>` then `aon admin reinit` |
| Sun not getting live Slack messages | `slack-bridge-sun` launchd not installed or dead | `launchctl list \| grep slack-bridge`; if missing see §4b above |
