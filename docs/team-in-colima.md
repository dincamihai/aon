# Team in Colima βÄî running an `aon` team inside the sandbox

Operator-facing guide for running a per-team agent fleet inside the
colima/AppArmor sandbox, with the operator on macOS host and one
interactive Claude Code session per role inside the VM. Pairs with
[`docs/sandbox.md`](sandbox.md) (sandbox internals) and the cmd-gate
ADRs ([001](adr/001-cmd-gate-layered-safety-gate.md),
[002](adr/002-self-evolving-classifier-prompt.md)).

> TL;DR. `cd ~/Repos/<team> && AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux --restart`
> One tmux session, one pane per role. Each pane = a live `claude`
> running as `ta-worker-<role>` inside the VM, persistent across ssh
> drops (via `dtach`), wired to the host's NATS broker, gated by
> cmd-gate's PreToolUse hook with sandbox-aware bypass.

## Architecture

```
Mac host                              colima VM (profile = "aon")
βîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄ                             βîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄβîÄ
 nats-server :4222 (host)             agents (claude TUI)
   βî€βîÄ sysadmin.creds (host-local)       βî€βîÄ ta-worker-rona     dtach @ /tmp/aon-rona.sock
   βîîβîÄ <role>.creds                      βî€βîÄ ta-worker-tim      dtach @ /tmp/aon-tim.sock
                                        βî€βîÄ ta-worker-sun      dtach @ /tmp/aon-sun.sock
 tmux session "<team>"                  βî€βîÄ ta-worker-mid      βÄΠ
   βî€βîÄ window "team":                    βî€βîÄ ta-worker-joana    βÄΠ
   βî²    panes attach via                βîîβîÄ ta-worker-ari      βÄΠ
   βî²    ssh -t βÄΠ dtach -a sock                  AppArmor enforce
   βî²                                            UIDs isolated, $HOME 0700
   βîîβîÄ window "security":                        per-role .claude.json (no sysadmin)
        aon security watch                      per-role NATS .creds
        (sysadmin creds, host)                  agents βÜ£ host.lima.internal:4222
```

Three trust zones, three boundaries:

| Zone | Lives on | Holds |
|---|---|---|
| Operator | macOS host | sysadmin creds, OAuth account, gate TUI |
| VM (root) | Linux VM | systemd units, AppArmor profiles, /etc/team-alpha |
| Agent | VM agent UID | own worktree, own creds, own .claude.json |

Each agent UID can read **only** its own creds + worktree. AppArmor +
DAC + per-role NATS ACL enforce.

## Components

### Host

| Path | Purpose |
|---|---|
| `~/Repos/ai-over-nats/scripts/sandbox/aon-tmux.sh` | engine launcher (team-agnostic) |
| `~/Repos/<team>/scripts/aon-tmux` | thin shim, delegates to engine, generated by `aon admin init` |
| `~/Repos/<team>/aon.toml` | roster + cmd-gate config |
| `~/.aon/teams/<team>/creds/<role>.creds` | per-role NATS JWT βÄî only that role's creds copied to VM |
| `~/.aon/teams/<team>/creds/sysadmin.creds` | operator NATS identity βÄî **never** copied to VM |
| `~/.config/slack-mcp/config.toml` | slack MCP auth βÄî pushed to `AON_SLACK_ROLES` (default sun) on each `--restart` |
| `~/.aon/colima-aon.ssh-config` | rendered colima ssh-config for `ssh -F` PTY use |

### VM

| Path | Purpose |
|---|---|
| `/work/workers/<role>/` | role's git worktree (cloned `--shared` from host's read-only mount on first start) |
| `/work/workers/<role>/.claude/settings.local.json` | per-role statusline + sonnet default |
| `/var/lib/team-alpha/workers/<role>/` | role's `$HOME` (0700) |
| `/var/lib/team-alpha/workers/<role>/.claude.json` | OAuth account metadata (sanitized: `installMethod=global-npm`, `projects` stripped) |
| `/var/lib/team-alpha/workers/<role>/.claude/.credentials.json` | OAuth tokens (copied from `ta-claude-auth` user βÄî see `aon admin claude-login`) |
| `/var/lib/ta-claude-auth/` | dedicated VM user holding the shared claude OAuth login |
| `/work/workers/<role>/.mcp.json` | per-role MCP config: aon + aon-board (+ slack for `AON_SLACK_ROLES`) |
| `/usr/local/bin/aon` | wrapper exec'ing host-mounted `$TA_HARNESS/bin/aon` |
| `/usr/local/bin/nats`, `/usr/local/bin/uvx` | downloaded binaries (apt only ships `nats-server`, no `uvx`) |
| `/etc/team-alpha/creds/<role>.creds` | NATS creds, 0600, ACL grants only that UID `r` |
| `/etc/team-alpha/env` | shared env: `AON_NATS_URL`, `TA_HARNESS`, etc. |
| `/etc/team-alpha/bypass` | sandbox bypass marker (root-only βÄî agent denied write by AppArmor) |
| `/tmp/aon-<role>.sock` | dtach socket; one live `claude` process bound to it |

### NATS topology (gate flow)

```
agent:  evt.coord-in.gate-request.<role>.<id>   βîÄβîÄβÜ£  operator
operator: evt.coord-out.gate-reply.<role>.<id>  βîÄβîÄβÜ£  agent
audit:  evt.security.gate.<role>                βîÄβîÄβÜ£  sysadmin (TUI / shadow)
```

Per-role ACL (auth template):
- `generalist` / `specialist`: pub on `gate-request.<own-role>.>`, sub on `gate-reply.<own-role>.>`. **Cannot** pub on reply path.
- `manager`: same plus other manager perms.
- `sysadmin`: wildcard `>` for both βÄî operator's identity.

Agent **cannot** self-approve, **cannot** impersonate peer.

## Daily flow

### Start the team (host terminal)

```bash
cd ~/Repos/<team>
AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux --restart
```

`--restart`: kills any stale dtach sessions in VM + tears down old host tmux. Fresh state every time. Drop the flag for normal "attach to running" runs.

What it does:
1. Render colima ssh-config to `~/.aon/colima-<profile>.ssh-config`.
2. For each role from the team's `aon.toml` roster:
   - Auto-create `ta-worker-<role>` user + `/work/workers/<role>` if missing (via `add-worker.sh`).
   - Push role's NATS creds to `/etc/team-alpha/creds/<role>.creds` (0600 + ACL).
   - Optional (`AON_SHARE_CLAUDE_AUTH=1`): scp the host's `~/.claude.json`, sanitize, drop into role's `$HOME/.claude.json`.
   - Run `start-agent-in-vm.sh <role>`:
     - First start: clones host's team repo (shared, hardlinks) into `/work/workers/<role>/`.
     - Writes `.claude/settings.local.json` with statusline + `model=sonnet`.
     - Spawns claude under `dtach -n /tmp/aon-<role>.sock` with full env (`AON_ROLE`, `AON_ROLE_KIND`, `AON_ROLE_DOMAIN`, `AON_NATS_URL=nats://host.lima.internal:4222`, `AON_CREDS=/etc/team-alpha/creds/<role>.creds`, `TERM=xterm-256color`, `COLORTERM=truecolor`).
     - Verifies dtach is alive after 0.5 s; bails loudly if claude crashed at startup.
3. Create tmux session `<team>` with two windows:
   - **`team`** βÄî tiled panes, one per role. Each pane runs `ssh -F <conf> -t <host> sudo -E -u ta-worker-<r> env TERM=$TERM dtach -a /tmp/aon-<r>.sock`. Detach a pane (close the ssh) βÜ£ claude keeps running in VM.
   - **`security`** βÄî single pane running `aon security watch` (textual TUI, sysadmin creds, host).
4. Attach to the session.

### Switch panes

| Keys | Action |
|---|---|
| `Ctrl-b 0` / `Ctrl-b 1` | jump to window team / security |
| `Ctrl-b o` | cycle pane within current window |
| `Ctrl-b βÜê/βÜ£/βÜë/βÜ™` | move to neighbor pane |
| `Ctrl-b z` | zoom current pane |
| `Ctrl-b d` | detach session βÄî agents keep running |

### Approve gate-asks

Window `security` shows the textual TUI. When an agent in the team window publishes a `gate-request`, it appears in the TUI:

| Key | Action |
|---|---|
| `βÜë`/`βÜ™`, `j`/`k` | navigate pending list |
| `y`, `Enter` | approve |
| `n` | deny |
| `r` | deny + reason (modal) |
| `q` | quit |

The approve / deny round-trips on NATS; the agent's blocked Bash unfreezes in the team-window pane.

### Detach for the night

```
Ctrl-b d
```

Agents stay alive in VM (dtach holds the claude processes; sockets persist). NATS broker on host stays running. Reattach next session:

```bash
cd ~/Repos/<team>
AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux
```

(no `--restart` βÜ£ ssh PTY reattaches to existing dtach sockets, conversations resume mid-thought.)

### Stop the team

```bash
./scripts/aon-tmux --restart                 # cleans state but immediately re-launches
```

Or to truly stop:

```bash
ssh -F ~/.aon/colima-aon.ssh-config -o ControlPath=none colima-aon \
  'sudo pkill -9 -f "dtach -n /tmp/aon-"; sudo rm -f /tmp/aon-*.sock'
tmux kill-session -t <team>
```

To stop everything including the VM:

```bash
colima stop --profile aon
```

VM persists between stops; restart with `colima start --profile aon`.

## One-time setup βÄî from scratch

### 0. Host prerequisites (macOS)

```bash
brew install colima lima docker          # vz hypervisor + docker CLI
brew install bash                        # bash >= 4 for aon
brew install jq nats-io/nats-tools/nats  # nats CLI for `aon security watch`
brew install nats-io/nats-tools/nsc      # only if you'll re-mint creds
brew install coreutils                   # gtimeout, used by nsc-smoke
```

Clone engine + create team:

```bash
mkdir -p ~/Repos && cd ~/Repos
git clone <ai-over-nats-url> ai-over-nats
pipx install --editable ./ai-over-nats        # exposes `aon` on PATH
mkdir <team> && cd <team>
aon admin init                                # bootstrap aon.toml
aon admin onboard rona                        # add a role (repeat per role)
aon admin reinit                              # mint creds + render prompts
```

NATS broker on host:

```bash
aon admin nats up                             # docker-compose nats:4222
```

### 1. Bring up VM (one-shot)

```bash
cd ~/Repos/ai-over-nats
TA_PROFILE=aon \
TA_HARNESS=$PWD \
TA_REPOS=$HOME/Repos \
TA_EXTERNAL_NATS=auto \
  bash scripts/sandbox/colima-up.sh
```

What this provisions inside the VM:
- apt: `acl apparmor auditd ca-certificates curl git jq nftables unzip nodejs npm nats-server systemd`
- nats CLI (download from github releases βÄî apt only ships server)
- uv / uvx (download from github releases βÄî needed by Python MCP servers)
- `/usr/local/bin/aon` wrapper that exec's the host-mounted `bin/aon`
- AppArmor profiles in enforce mode
- systemd units for `team-alpha-coord` + `team-alpha-worker@<role>`
- `/etc/team-alpha/env` with `AON_NATS_URL=nats://host.lima.internal:4222`

Mounts (declared in `colima-up.sh`):

| Host path | VM mode | Purpose |
|---|---|---|
| `$TA_HARNESS` (`ai-over-nats`) | RO | engine source for in-VM `aon` wrapper |
| `$TA_REPOS` (`~/Repos`) | RO | team worktrees, slack-mcp source |
| `$TA_AON_BOARD` (`~/aon-board`) | RW | board MCP state βÄî auto-detected if dir exists |
| `~/.team-alpha/apparmor` | RO | personal AppArmor overrides |

`TA_EXTERNAL_NATS=auto` sets `AON_NATS_URL=nats://host.lima.internal:4222` so VM agents reach the host broker; in-VM `team-alpha-nats` unit is masked. Operator's `sysadmin.creds` stays on host.

Verify:

```bash
colima ssh --profile aon -- bash -c 'which aon nats uvx; mount | grep Users'
```

### 2. Install per-team launcher shim (per team)

```bash
cd ~/Repos/<team>
aon admin init                                    # idempotent; drops scripts/aon-tmux shim
git add scripts/aon-tmux && git commit -m "..."   # share with team
```

### 3. Seed claude OAuth in VM (one-time)

Avoids login-per-role + the macOS Keychain detour. Logs in once as a dedicated VM user; aon-tmux copies the resulting tokens into each role's home on each `--restart`.

```bash
aon admin claude-login                            # interactive βÄî complete OAuth, /exit
```

Re-run only when host token rotates or the VM is recreated.

### 4. (Optional) Slack MCP for sun

Slack MCP runs only for roles listed in `AON_SLACK_ROLES` (default: `sun`). Source lives at `/Users/mid/Repos/slack-mcp` (mounted RO in VM). Auth config gets pushed from host on each `aon-tmux` run:

```bash
# host: configure slack-mcp once
mkdir -p ~/.config/slack-mcp
cat > ~/.config/slack-mcp/config.toml <<EOF
# tokens, channels, etc. βÄî see /Users/mid/Repos/slack-mcp/config.example.toml
EOF
```

`scripts/sandbox/aon-tmux.sh` `share_slack_config()` scp's it into `/var/lib/team-alpha/workers/sun/.config/slack-mcp/config.toml` automatically. To grant slack to other roles:

```bash
AON_SLACK_ROLES="sun mid" ./scripts/aon-tmux --restart
```

### 5. Configure cmd-gate evolve (optional)

In the team's `aon.toml`:

```toml
[security.cmd_gate.evolve]
backend_provider = "ollama"
model            = "qwen3.5:9b"
budget_usd       = 0
```

For Anthropic judge instead, set `backend_provider = "anthropic"` and `ANTHROPIC_API_KEY` in env.

### 6. (Optional) Personal AppArmor overrides

See [`docs/sandbox.md` ¬ß Personal AppArmor overrides](sandbox.md). Drop additional allow/deny rules in `~/.team-alpha/apparmor/` and re-run `colima-up.sh`.

### 7. First launch

```bash
cd ~/Repos/<team>
AON_SHARE_CLAUDE_AUTH=1 ./scripts/aon-tmux --restart
```

In each pane: `/mcp` lists `aon`, `aon-board` (and `slack` for sun). Statusline shows `<role> - <kind>`.

## Per-role MCP wiring

`start-agent-in-vm.sh` writes `$work/.mcp.json` after cloning the team repo. All roles get `aon` + `aon-board`. Roles in `$AON_SLACK_ROLES` (default `sun`) additionally get `slack` via `uvx --from /Users/mid/Repos/slack-mcp slack-mcp`.

To customize, edit the `mcp_extra` block in `start-agent-in-vm.sh`. Per-role conditional MCPs follow the same pattern.

## Trust model recap

The gate is **intent gating** βÄî argv-level. Defense in depth:

| Layer | Stops |
|---|---|
| `cmd-gate` deny.regex hard floor | `rm -rf /`, `git push --force`, `terraform destroy`, etc. βÄî even with bypass |
| `cmd-gate` classifier | hidden flags, base64 obfuscation, char-code SQL, etc. |
| `cmd-gate` operator-ask | borderline argv βÄî operator approves via TUI |
| AppArmor `team-alpha-worker` | per-UID FS scope, syscall filter, deny `/etc/`, deny peer worktrees |
| DAC (Linux UID + 0700 dirs) | each agent reads/writes only own home + worktree |
| NATS ACL (per-role JWT) | agent can't impersonate peer or self-approve |
| host broker outside VM | sysadmin creds untouchable from VM (kernel boundary) |
| colima VM (vz/aarch64) | kernel boundary; agent escape doesn't reach host |

What this **doesn't** defend:

- Anthropic API session: VM-side `ta-claude-auth` OAuth shared across roles βÜ£ all agents bill to one account; concurrent session limits may apply.
- Network egress: agent in VM can reach any host its iptables allows. Add `nft` rules per [`docs/sandbox.md` ¬ß What this does not do](sandbox.md).
- Kernel exploits: VM is the last line. Agents are kernel-isolated from host, but exploit + escape gets you a VM-level breach (host still safe).

| Env / file | Effect |
|---|---|
| `AON_SHARE_CLAUDE_AUTH=1` | copy claude OAuth from `ta-claude-auth` VM user to each role (default on; set =0 to disable) |
| `AON_SLACK_ROLES="sun mid"` | space-separated list of roles that get slack MCP (default: `sun`) |

| `AON_TMUX_SESSION=<name>` | tmux session name (default = team basename) |
| `TA_AON_BOARD=<path>` | host dir mounted RW for board MCP state (default: `~/aon-board` if exists) |
| `~/.team-alpha/apparmor/{base,coord,worker}` | personal AppArmor overrides synced into VM |
| `/etc/team-alpha/bypass` | root-only marker; flips classifier bypass for ALL roles in this VM |
| `aon.toml [security.cmd_gate]` | classifier model, timeouts, fallback |
| `aon.toml [security.cmd_gate.evolve]` | judge backend + model + budget |

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `can't find window: team` | dtach socket exists but claude crashed | `./scripts/aon-tmux --restart` (start-agent now detects + clears orphans) |
| `Not logged in ¬Ζ Run /login` in every pane | host `~/.claude.json` not shared, or share truncated, or installMethod mismatch | run with `AON_SHARE_CLAUDE_AUTH=1`; verify VM file is full size (~92 KB) |
| Mono colors (kitty terminal) | `kitty term=xterm-kitty` doesn't propagate | set `term xterm-256color` in `~/.config/kitty/kitty.conf` |
| `unknown shorthand flag: t in -t` | tried `colima ssh -t` βÄî colima ssh has no PTY flag | use `ssh -F ~/.aon/colima-<profile>.ssh-config -t colima-<profile>` |
| `Inappropriate ioctl for device` from claude | TERM=unknown bled through | `--restart`; the dtach env now hardcodes TERM=xterm-256color |
| `Session open refused by peer` / `exit 255` | stale ssh control socket | `rm ~/.colima/_lima/colima-aon/ssh.sock`; `aon-tmux` does this automatically each run |
| `agents/ missing` from `aon doctor` | A2A discovery cards never populated | `mkdir ~/Repos/<team>/agents` (cosmetic βÄî only matters if you use A2A skill routing) |
| Gate denies a legitimate command | `deny.regex` is hard-floor (irreversible) | use `~/.aon/security/allow.local.regex` to whitelist the pattern; or `aon security bypass on` for short-lived dev (still respects deny.regex) |
| Operator TUI shows nothing on agent's gate-request | per-role NATS perms stale | re-mint role JWT: `nsc delete user --account <team> --name <role>` then `aon admin reinit` |
